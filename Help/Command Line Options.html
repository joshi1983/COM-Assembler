Command Line Options

Overview

If you want to use another code editor or assemble code from your own programs, you can create a batch file that contains DOS commands that can tell the assembler what to do.
Caution - watch out for using spaces in your file names. They parse the parameters. I suggest using the underscore, '_', character instead.

Options

  • Assembly Process Document - You can set an option to create a text file that contains information on how the source file was assembled using the makeprocdoc command.
  • Error List Files - To set the file for outputting the error list, use the -e command. Follow this by the name of your text file.
  • Load File - Load a file into the GUI using the -l command. The following extentions will work: *.com; *.exe; *.txt; *.asm; *.inc
  • No Close - if you specify the output file, the program will usually close just after creating the output files. If you want the assembler to stay running, use the noclose command.
  • Output - Specify the output program file using -o or just by giving the com program's name. If you use the -e command, put the name of the program after it.
  • Source - Specify the source assembly file using -s or just by listing the assembly source file. This is to be followed by the name of your source file.
